## Stow Architecture

Two separate stow dirs, both targeting `$HOME`:

```
dotfiles/                          # stow dir for base packages
  hypr/ kitty/ waybar/ rofi/
  dunst/ tofi/ wlogout/ btop/
  gtk/ assets/
themes/                            # stow dir for theme packages
  .stow                            # marks this as its own stow boundary
  catppuccin/
```

**Always stow with `--no-folding`** — theme and base packages share `.config/<tool>/` directories. Without `--no-folding`, stow creates a single directory symlink that blocks the other package from adding files into the same dir.

```bash
# Apply base packages
stow --dir=~/dotfiles --target=$HOME --no-folding <package>

# Apply a theme
stow --dir=~/dotfiles/themes --target=$HOME --no-folding catppuccin

# Remove a theme
stow --dir=~/dotfiles/themes --target=$HOME -D catppuccin
```

## Base vs Theme Split

Each tool's config is split:

| Package | What stays in base | What goes in theme |
|---|---|---|
| hypr | hyprland.conf, hypridle.conf, monitors, workspaces, autostart | theme.conf (border colors, wallpaper var), hyprlock.conf |
| kitty | kitty.conf | theme.conf (colors) |
| waybar | config / config.jsonc | style.css |
| dunst | — | dunstrc (full file) |
| tofi | — | configA, configV (full files) |
| wlogout | layout | style.css |
| btop | btop.conf (references theme by name) | themes/catppuccin-macchiato.theme |
| gtk | gtk.css, settings.ini, window_decorations.css, assets/ | colors.css |
| rofi | template/, non-theme rasi files | catppuccin.rasi |

## Hyprland-specific Quirks

**Race condition**: Hyprland rewrites `hyprland.conf` with an autogenerated stub while running. After any stow operation that touches the hypr package, use `--adopt` to let stow take the stub, then verify the dotfiles version still has the real config (check for `source =` at line ~20).

**Source path**: `hyprland.conf` sources the active theme via hardcoded absolute path — NOT through the symlink:
```
source = /home/alban/dotfiles/themes/catppuccin/.config/hypr/theme.conf
```
Hyprland's glob engine fails on symlinks, so the source must point directly into the dotfiles tree. When switching themes, update this line.

**Reload after stow**: Stow changes don't auto-apply. Run `hyprctl reload` after any hypr config change.

## Adding a New Theme

1. `mkdir -p ~/dotfiles/themes/<name>/.config/`
2. Copy the per-tool theme files listed in the table above into the new theme dir
3. Update the `source =` line in `hypr/.config/hypr/hyprland.conf` to point to the new theme
4. `stow --dir=~/dotfiles/themes --target=$HOME -D catppuccin`
5. `stow --dir=~/dotfiles/themes --target=$HOME --no-folding <name>`
6. `hyprctl reload`
